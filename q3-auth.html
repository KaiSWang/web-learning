Password Storage · 密碼儲存

正確的密碼
絕對不能直接存

新手常犯的最大錯誤：把使用者密碼直接存進資料庫。一旦資料庫外洩，所有人的密碼都被看光—— 而且很多人會在多個網站用同一組密碼，傷害會擴散到其他網站。

▸ 密碼儲存的演進
業界標準的密碼 Hash 演算法
  • bcrypt最廣泛使用，調整 cost factor 來增加運算成本
  • argon22015 密碼 hash 競賽冠軍，新專案推薦
  • scrypt記憶體成本高，抗 GPU 暴力破解
  • PBKDF2較舊但仍合規，常見於 PCI-DSS、FIPS 場景

2FA · 雙因素驗證

2FA 為什麼
能擋住大部分攻擊？

認證的「因素」分三類：你知道的（密碼）、你有的（手機/實體 key）、你是的（指紋/臉）。 只用密碼是單因素，駭客拿到帳密就能登入。2FA 至少要兩種，大幅提高難度。

A

SMS OTP · 簡訊驗證碼

最常見也最弱的 2FA。電信業者可能被社工、SIM swap 攻擊（駭客把你的門號補卡到他的 SIM）會繞過。 適合一般網站，但金融、加密貨幣等高價值帳號不建議只用這個。

B

TOTP · 時間型一次性密碼

Time-based One-Time Password

就是 Google Authenticator、Authy 用的方式。原理：伺服器和你的 App 共享一個 secret， 雙方根據當前時間 + secret各自算出一個 6 位數，每 30 秒換一次。 駭客即使看到你輸入的數字，30 秒後也失效。

# 設定時：伺服器產生 secret，給你掃 QR code
secret = JBSWY3DPEHPK3PXP
QR Code = otpauth://totp/Service:KS?secret=JBSW...

# 使用時：雙方各自算
code = HMAC-SHA1(secret, current_time / 30)
      = "847291"
C

Hardware Key · 實體金鑰

YubiKey、Google Titan 這類 USB 裝置。最強的 2FA——金鑰永遠不離開硬體， 駭客即使遠端控制你的電腦也無法盜用。Google 員工強制使用，內部多年沒發生釣魚成功的案例。

D

Passkey · 無密碼登入

2024 開始普及的下一代方案。基於 WebAuthn 標準。 用 Face ID / Touch ID / Windows Hello 取代密碼，背後用公私鑰加密。 私鑰存在你的裝置（不可匯出），網站只存公鑰——伺服器被駭也無法登入。

Passkey 的優勢
  • 不可釣魚金鑰綁定特定網域
  • 不可暴力破解沒有密碼可猜
  • 跨裝置同步用 iCloud Keychain / Google Password Manager
  • 使用者體驗一次生物辨識搞定
