Notion 飛機上也能寫，
Gmail 沒網路還能讀信

傳統認知：網頁 = 要連網才能用。但 Notion、Gmail、Google Docs 即使離線都還能繼續操作， 連上網後再自動同步。這背後是 PWA（Progressive Web App）的能力—— 讓網頁擁有「像 App 一樣」的體驗。

Three Pillars · 三大支柱

PWA 靠三項技術實現離線

離線網頁的核心是：把該網頁的所有資源（HTML、CSS、JS、API 回應、使用者資料） 存在瀏覽器內的儲存空間，沒網路時從本地讀。三大關鍵技術各司其職。

▸ PWA 架構：Service Worker 是核心
Service Worker · 攔截器詳解

Service Worker 是
網頁的「網路代理人」

Service Worker 是一個獨立的 JavaScript 執行緒，跟主網頁分開跑。 它最特別的能力：攔截網頁所有的網路請求，可以決定要走網路還是回傳快取。 這就是 PWA 離線運作的核心。

A

註冊與生命週期

網頁第一次載入時註冊 Service Worker。它會獨立於網頁存在——使用者關閉瀏覽器分頁，它仍然背景待命， 這也是為什麼能做推播通知。

# 主網頁
if ('serviceWorker' in navigator) {
  navigator.serviceWorker.register('/sw.js');
}
B

攔截 fetch · 三種策略

一旦 Service Worker 啟動，所有 fetch 請求都會先經過它。它可以決定：

# 策略 1：Cache First（快取優先）— 適合靜態資源
self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request).then(cached =>
      cached || fetch(event.request)
    )
  );
});

# 策略 2：Network First（網路優先）— 適合 API 資料
fetch(event.request).catch(() => caches.match(event.request))

# 策略 3：Stale While Revalidate（先回快取再背景更新）
# 使用者立刻看到內容，背景同時抓最新版
C

背景同步 · 離線編輯後自動上傳

離線時使用者修改文件，把變動寫進 IndexedDB； 連網後 Service Worker 偵測到網路恢復，自動把佇列中的修改上傳到伺服器。 這是 Notion 離線編輯能無縫同步的關鍵。

# 註冊背景同步
navigator.serviceWorker.ready.then(reg => {
  reg.sync.register('sync-pending-edits');
});

# Service Worker 內處理
self.addEventListener('sync', (event) => {
  if (event.tag === 'sync-pending-edits') {
    event.waitUntil(uploadPendingEdits());
  }
});
D

推播通知 · 像原生 App

Service Worker 還能接收伺服器推送的通知，即使網頁沒開——這就是為什麼 Web 版 LINE、Twitter 能跳通知。

Manifest · 安裝到主畫面

讓網頁變成 App 圖示

除了離線能力，PWA 還能讓使用者把網站「裝」到手機主畫面，打開時全螢幕、沒有瀏覽器網址列—— 跟原生 App 看起來幾乎一樣。這需要一個 manifest.json 檔。

# manifest.json
{
  "name": "Web Atlas",
  "short_name": "Atlas",
  "start_url": "/",
  "display": "standalone",  # 全螢幕
  "theme_color": "#8b3a1e",
  "background_color": "#faf7f2",
  "icons": [
    { "src": "/icon-192.png", "sizes": "192x192" },
    { "src": "/icon-512.png", "sizes": "512x512" }
  ]
}
PWA vs 原生 App
  • 不用上架不用 App Store 審核（你做 sinoApp 應該感同身受）
  • 跨平台一份程式碼，iOS/Android/桌機都能裝
  • 更新即時使用者不用手動更新
  • 受限的能力無法完全使用某些原生 API（藍牙、HealthKit 等）
  • iOS 限制較多Apple 對 PWA 的支援度低於 Android
Real Examples · 經典案例

哪些網站偷偷是 PWA？

網站 離線能做什麼 用了什麼技術
Notion 讀取、編輯文件，回網後同步 Service Worker + IndexedDB + 衝突解決
Gmail 讀過去 90 天的信件、寫草稿 Service Worker + Cache API
Twitter Lite 瀏覽快取的 timeline Service Worker + App Shell 模式
Spotify Web 離線播放下載過的歌 Service Worker + 加密的 IndexedDB
Google Maps 離線地圖瀏覽 Cache API + 大量本地圖磚
