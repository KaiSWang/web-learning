網頁設計的
六大領域全貌

在我們深入「一次點擊背後發生什麼」之前，先看整個領域的樣貌。 網頁世界由六大層次組成，每一層解決不同的問題、由不同的角色負責。 理解這張地圖，你就知道每個技術名詞屬於哪一塊、彼此怎麼合作。

Six Pillars · 逐塊解析

六大核心領域逐一拆解

上面圖中的每一塊，都是一個獨立的學科。這裡用最白話的方式介紹，讓你建立基本概念。 後續章節會深入每一塊的細節。

①

UX · 使用者體驗

User Experience · 為誰、解決什麼問題

一切的起點。在動筆畫畫面之前，先弄清楚：使用者是誰？他要完成什麼？目前的痛點在哪？ 這層關心的是「對的問題」，而不是「漂亮的解法」。

舉例：你在設計一個股票交易 App，UX 階段你要先想——使用者買賣時最焦慮什麼？ 是怕點錯按鈕？還是怕看不到行情變化？這會影響後續所有設計決策。

這個領域的關鍵概念
  • 使用者研究訪談、問卷、行為觀察
  • Persona虛構一個典型使用者，幫助團隊聚焦
  • User Flow使用者完成任務的步驟流程圖
  • Wireframe線框稿，只畫結構不畫細節
  • IAInformation Architecture，資訊架構（頁面怎麼分層）
②

UI · 視覺設計

User Interface · 看起來什麼樣

把抽象的流程變成具體的畫面。處理排版、色彩、字體、間距、圖示這些視覺語彙， 讓介面不只能用，還要好看且有個性。

這個領域的關鍵概念
  • 排版系統Grid System、Spacing Scale
  • 色彩系統主色、輔色、語意色（成功/警告/錯誤）
  • 字體系統Typography Scale，標題、內文、註解的層次
  • Design System把所有設計規則文件化、元件化（如 Material Design）
  • Figma業界標準設計工具，類似工程界的 VS Code
③

HTML · 結構層

HyperText Markup Language · 「這是什麼」

網頁的骨頭。HTML 描述「這是標題、這是段落、這是按鈕」，重點在語意正確。 HTML 不負責長相，只負責定義「這個東西是什麼」。

# 一個按鈕的 HTML
<button type="submit">送出</button>

# 一段標題
<h1>網頁設計學習地圖</h1>
這個領域的關鍵概念
  • 語意標籤<header>、<nav>、<article> 等有意義的標籤
  • 表單<form>、<input>、<select> 收集使用者輸入
  • a11yAccessibility，無障礙（讓視障者用螢幕閱讀器也能用）
  • SEOSearch Engine Optimization，搜尋引擎優化（Google 怎麼讀懂你的頁面）
④

CSS · 樣式層

Cascading Style Sheets · 「長什麼樣」

負責「長什麼樣子」。CSS 處理顏色、尺寸、版面、響應式（手機/平板/桌機自動適應）、動畫、過場效果。 這是初學最容易卡關的地方，因為視覺直覺與程式邏輯經常不一致。

# 把按鈕變成橘色
button {
background: #8b3a1e;
padding: 12px 24px;
border-radius: 4px;
}
這個領域的關鍵概念
  • Flexbox / Grid兩大版面系統（一維 / 二維）
  • RWDResponsive Web Design，響應式設計（一份程式碼適應所有螢幕）
  • Animation / Transition動畫與過場
  • Tailwind CSS主流的工具集 CSS 框架（Class 名稱直接組合出樣式）
  • Sass / LessCSS 預處理器（給 CSS 加上變數、巢狀等程式特性）
⑤

JavaScript · 互動層

JS · 「會做什麼」

讓網頁「會動」。點擊、輸入、跳出視窗、跟伺服器拿資料、即時更新內容——都是 JS 的工作。 進階會走到 React / Vue 這類框架，把複雜的互動結構化。 對你（韌體工程師）而言，這層的思維最熟悉——事件驅動、狀態管理、非同步，跟 RTOS 概念很像。

這個領域的關鍵概念
  • DOMDocument Object Model，瀏覽器把 HTML 解析成的物件樹，JS 操作的對象
  • Event事件（點擊、滑動、鍵盤輸入）
  • API / AJAX / Fetch跟伺服器要資料
  • React / Vue / Svelte主流前端框架
  • TypeScriptJS 的進階版，加上型別檢查（韌體背景的人通常很愛）
⑥

部署 · 讓世界看到

Deployment & Infrastructure

把寫好的網頁放上網路。涉及網域、DNS、HTTPS、CDN、託管平台、效能優化。 這層的目標：快、穩、安全。 這部分你已經有實戰經驗（sino-suisse 的 Cloudflare 遷移）。

這個領域的關鍵概念
  • DNS網域名稱轉 IP 的系統
  • HTTPS / TLS加密通訊
  • CDNContent Delivery Network，內容傳遞網路（Cloudflare、Akamai）
  • 託管平台Vercel、Netlify、Cloudflare Pages、AWS
  • CI/CDContinuous Integration / Deployment，自動化部署
Roles · 業界角色分工

真實團隊裡
誰負責哪一塊？

上面六塊在小公司可能一個人全包，在大公司則由不同角色分工。理解角色分工，能幫你判斷自己想成為哪種工程師。

▸ Web 團隊角色光譜

越靠左越接近設計與使用者，越靠右越接近基礎設施

以你的背景（韌體工程師、系統思維），最自然的延伸是後端工程師或DevOps， 因為這兩塊跟系統設計、底層協定的距離最近。前端工程師的視覺直覺成本對你會比較高，但若你想做個人專案、freelance 接案，前端是必修。
