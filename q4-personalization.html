Q · 04 · Personalization

你和朋友打開同一個 YouTube，
為什麼看到完全不同？

網址都是 youtube.com 首頁，HTML 模板也一樣，但內容卻是「為你客製」。 Netflix、IG、Amazon 都有這種能力。背後是前後端分工 + 推薦演算法 + A/B 測試的整合系統。

Separation · 內容與容器分離

「畫面框架」是同一份，
「內容」每人不同

個人化的關鍵在於：HTML 模板（容器）對所有人一樣，但填進容器的資料由後端依使用者動態決定。

▸ 同一個網址產生不同內容的機制
Algorithms · 推薦演算法基礎

系統怎麼知道你會喜歡？

推薦演算法是個複雜的領域，但核心概念可以分成三大類。實務中通常是混合使用。

①

Content-based · 基於內容

「你看過 A，A 跟 B 很像，所以推 B」

分析每個項目的特徵（電影類型、演員、關鍵字），把使用者過去喜歡的特徵組成一個「興趣向量」。 找特徵向量相似的新項目推薦給你。

# 簡化的特徵向量
影片 A: { 科技: 0.9, 韌體: 0.8, 教學: 0.7 }
影片 B: { 科技: 0.8, 嵌入式: 0.9, 教學: 0.6 } ← 高相似度
影片 C: { 音樂: 0.9, 流行: 0.8 } ← 低相似度

# 你看過 A → 推 B 而不是 C
②

Collaborative Filtering · 協同過濾

「跟你品味相似的人都喜歡 X」

不分析內容本身，而是找出「跟你行為模式相似的其他使用者」，把他們喜歡但你沒看過的推給你。 Netflix、Amazon 起家就是用這個。

# 使用者-項目矩陣
        影片1 影片2 影片3 影片4
KS    ★    ★    ?     
小明 ★    ★    ★    # 跟 KS 看的一樣
小華 ★        ★    ★

# 推給 KS 影片3（小明也看了）
③

Deep Learning · 深度學習

現代主流：YouTube、TikTok 用的

把使用者特徵、項目特徵、上下文（時間、裝置、地點）全部丟進神經網路，學出一個「誰會點什麼」的模型。 TikTok 的 For You 演算法就是這類，能在你看 3-5 部影片後就抓到偏好。

這需要大量資料和運算資源，通常只有大公司能負擔。中小型網站還是以協同過濾為主。

A/B Testing · 怎麼驗證有效

新功能上線前，
怎麼知道有沒有用？

不能憑感覺上線。現代網站會把 5% 使用者隨機分到「新版」、其他人留在「舊版」， 跑一兩週看數據——這叫 A/B Test。

▸ A/B Test 運作方式

你之前用過的 AlphaTrader 平台、Netflix 都是這樣運作。Google 一年跑數萬個 A/B Test， 連按鈕顏色、字級這種小變化都會測。直覺常常騙人，數據才是真的。
